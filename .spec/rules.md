# Rules

## Never
- Run `npx stack` in this repo (test via `pnpm link` in app repos)
- `git add/commit/push` without user approval
- Delete/modify existing code comments without asking
- Put SSH calls in fix functions
- Put `GITHUB_ACTIONS` checks in fix functions
- Use template literals in YAML string generation (breaks YAML indentation)
- Write bash >5 lines in workflow files

## Always
- Check STANDARDS.md before modifying plugin/workflow/architecture code
- Warn on standards violations before proceeding
- Route via `canReach()` — fixes are local-only, pipeline handles remote
- Use string concat for YAML values: `'key: ' + var`
- Mark critical code with `// CRITICAL:` block (Why + Breaks-if-changed)
- Include `# Generated by @factiii/stack vX.X.X` in generated files
- Split plugins >1000 lines into index.ts + scanfix/ + environment files

## Commands are dumb
`scan.ts`, `fix.ts`, `deploy.ts` know nothing about GITHUB_TOKEN, SSH, or workflows. All routing lives in pipeline plugins.
