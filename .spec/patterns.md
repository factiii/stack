# Code Patterns

## File Generation
```
1. Check if file exists
2. If exists, compare content
3. Skip if unchanged (avoid unnecessary writes/git noise)
4. Write with header: # Generated by @factiii/stack vX.X.X
```

## YAML String Building
```typescript
// CORRECT — string concat
const yaml = 'server: ' + envConfig.server + '\n' +
  'domain: ' + envConfig.domain + '\n';

// WRONG — template literals break YAML indentation
const yaml = `server: ${envConfig.server}
domain: ${envConfig.domain}`;
```

## Config Value Conventions
```
Required values:  prefix with EXAMPLE-  (e.g., domain: EXAMPLE-myapp.com)
Auto-detected:    use OVERRIDE pattern   (user can override auto-detected values)
```

## SSH Key Convention
```
Key path:    ~/.ssh/{stage}_deploy_key
Vault name:  {STAGE}_SSH  (e.g., PROD_SSH, STAGING_SSH)
Secret name: {STAGE}_{TYPE} format (PROD_SSH not SSH_PROD)
```

## Critical Code Blocks
```typescript
// CRITICAL: Why this exists
// Breaks-if-changed: What goes wrong
const importantThing = ...;
```

## OS-Specific Fixes
```typescript
{
  id: 'mac-only-fix',
  os: 'mac' as ServerOS,  // cast required
  // ...
}
```

## Import Style
```typescript
import type { Fix, FactiiiConfig } from '../../types/index.js';  // .js extension required (ESM)
```

## Plugin File Structure (>1000 lines)
```
plugins/{category}/{name}/
  index.ts      — canReach(), fixes[], commands[], deployStage()
  scanfix/      — grouped fix files (bootstrap.ts, secrets.ts, etc.)
  workflows/    — generated workflow templates (pipeline only)
  utils/        — helper functions
```
